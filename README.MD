# 멀티모달 피부 코치 프로젝트 설명서

## 1. 이 프로젝트를 한 줄로 설명하면

이 프로젝트는 **"얼굴 사진만 보는 단순 여드름 분류기"**가 아니라,
**"피부 사진 + 생활습관 기록 + 시간 흐름"을 함께 보고 피부가 왜 나빠졌는지 또는 왜 좋아졌는지를 설명해주는 멀티모달 피부 코치 시스템**입니다.

쉽게 비유하면,

- 사진 모델은 `피부 시험지를 채점하는 선생님`
- 시계열 모델은 `생활습관 일기장을 읽는 탐정`
- 멀티모달 모델은 `선생님과 탐정의 의견을 합쳐 이유를 설명하는 담임선생님`

이라고 생각하면 됩니다.

즉, 이 시스템은 단순히

- "여드름 있음"

만 말하는 것이 아니라,

- 지금 피부 상태가 어떤지
- 최근 2주~1개월 동안 좋아지는 중인지, 나빠지는 중인지
- 무엇이 원인 후보인지
- 어떤 생활습관이 영향을 줬는지
- 어떤 피부 요소가 특히 나빠졌는지

를 함께 설명하는 것이 목표입니다.

---

## 2. 왜 "단순 여드름 분류기"로는 부족한가

피부는 시험 점수처럼 한 과목만 있는 게 아닙니다.

예를 들어 어떤 사람은

- 여드름은 줄었는데
- 건조함은 심해지고
- 붉은기는 그대로이며
- 색소침착은 더 눈에 띌 수 있습니다.

즉 피부는 `수학 한 과목 점수`가 아니라,
`수학, 영어, 과학, 체육 점수를 다 같이 보는 성적표`에 가깝습니다.

그래서 이 프로젝트는 여드름만 보는 모델이 아니라, 최소한 아래 요소들을 함께 봐야 합니다.

- 여드름
- 주름
- 건조
- 붉은기
- 색소침착
- 유분
- 모공
- 민감도 관련 시각 특징

정리하면,
이 프로젝트의 핵심은 **"한 장의 사진을 맞히는 모델"**이 아니라
**"피부 변화의 이유를 시간 흐름과 함께 해석하는 시스템"**입니다.

## 3. 지금 개발 범위

현재 개발 범위는 아래 3개 모델까지입니다.

### A. 이미지 분석 멀티헤드 모델

얼굴/피부 사진 한 장을 보고 여러 피부 상태를 동시에 점수화합니다.

### B. 시계열 원인 분석 모델

최근 며칠~몇 주 동안의 생활습관, 제품 사용, 환경 변화, 피부 점수 변화를 보고 악화/개선 원인을 추정합니다.

### C. 멀티모달 원인 추론 모델

이미지 결과와 시계열 결과를 합쳐서,
왜 피부가 변했는지 최종 설명을 만듭니다.

중요:

- 지금 단계에서는 **제품 추천 엔진**과 **퍼스널 컬러 추천 엔진**은 본 개발 범위 밖의 다음 단계입니다.
- 하지만 앞으로 확장될 수 있도록 README에는 연결 구조까지 설명합니다.

- 나중에 성능이 안정되면 Transformer 계열로 확장하면 됩니다.

## 5. 모델 전체를 쉬운 비유로 설명하면

이 프로젝트 전체는 `피부 병원의 디지털 팀`처럼 생각하면 쉽습니다.

### 5-1. A 모델: 이미지 분석 멀티헤드 모델

이 모델은 `사진을 보고 피부 성적표를 만드는 채점 선생님`입니다.

학생 사진을 보고 선생님이

- 국어 몇 점
- 영어 몇 점
- 수학 몇 점

을 따로 채점하듯,
이 모델은 사진 한 장에서 여러 피부 항목을 동시에 채점합니다.

예를 들면 아래처럼 점수를 냅니다.

- 여드름 점수
- 붉은기 점수
- 건조 점수
- 주름 점수
- 색소침착 점수
- 유분 점수
- 모공 점수

여기서 "멀티헤드"는
`한 몸에 여러 개의 채점기`가 달린 것과 같습니다.

즉 사진은 하나지만,
출력 창구는 여러 개인 구조입니다.

### 5-2. B 모델: 시계열 원인 분석 모델

이 모델은 `생활습관 일기를 읽는 탐정`입니다.

예를 들어 최근 14일 동안 아래 정보가 있다고 해봅시다.

- 수면 시간
- 물 섭취량
- 당류 섭취
- 유제품 섭취
- 마스크 착용 시간
- 스트레스 점수
- 햇빛 노출
- 운동량
- 세안 습관
- 제품 변경 여부

탐정은 이 기록을 보고
"피부가 나빠진 날 전부터 무엇이 달라졌지?"
"좋아진 날에는 어떤 습관이 유지됐지?"
를 찾습니다.

즉 이 모델은 `시간 흐름 속 패턴`을 읽습니다.

### 5-3. C 모델: 멀티모달 원인 추론 모델

이 모델은 `채점 선생님 + 탐정`의 의견을 합쳐 최종 보고서를 쓰는 담임선생님입니다.

예를 들어,

- 사진상 붉은기와 여드름 점수가 떨어졌고
- 지난 10일 동안 수면 부족, 스트레스 증가, 새 제품 사용이 있었고
- 햇빛 노출도 늘었다면

이 모델은
"최근 피부 악화는 수면 부족, 스트레스 증가, 제품 변경의 영향이 함께 작용했을 가능성이 큽니다."
처럼 설명합니다.

중요한 점은,
이 모델은 보통 **원인 하나만 단정하지 않고**
`원인 후보들의 기여도`를 보여주는 방향이 더 안전합니다.

## 6. "피부가 나빠졌다 / 좋아졌다"를 어떻게 판단할 것인가

사용자 요청에서 가장 중요한 요구사항 중 하나는 이것입니다.

- 피부 점수가 낮아졌다면: 왜 나빠졌는지 설명
- 피부 점수가 높아졌다면: 왜 좋아졌는지 설명

이 기능은 아래처럼 설계하는 것이 가장 자연스럽습니다.

### 6-1. 사용자에게 보여줄 점수는 "높을수록 좋게" 설계

사용자 입장에서는 `높을수록 좋은 피부 점수`가 이해하기 쉽습니다.

예:

- 피부 종합 점수: 82점
- 여드름 점수: 76점
- 붉은기 점수: 68점
- 건조 점수: 59점
- 색소침착 점수: 73점

즉 점수가 높을수록 상태가 좋은 방향으로 맞춥니다.

### 6-2. 내부적으로는 두 종류의 점수를 가질 수 있음

실제로 모델 내부에서는 아래 2개를 나눠 가질 수 있습니다.

1. `증상 심각도 점수`
   - 높을수록 나쁨
   - 예: 여드름 심각도 80

2. `사용자 표시용 건강 점수`
   - 높을수록 좋음
   - 예: 여드름 건강 점수 20

앱에서는 혼란을 줄이기 위해
**최종적으로는 "건강 점수"만 보여주는 방식**이 좋습니다.

### 6-3. 악화/개선 판단 방식

예를 들어 오늘 점수와 2주 전 점수를 비교합니다.

- 오늘 피부 종합 점수 61
- 2주 전 피부 종합 점수 74

이 경우 `13점 하락`이므로 피부가 악화된 것으로 판단합니다.

반대로

- 오늘 피부 종합 점수 79
- 2주 전 피부 종합 점수 65

이면 `14점 상승`이므로 피부가 개선된 것입니다.

### 6-4. 설명 문장은 어떻게 만들까

설명은 무조건 `점수 변화 + 나빠진(또는 좋아진) 세부 요소 + 원인 후보 기여도`를 함께 보여주는 것이 좋습니다.

예시 1. 나빠진 경우

- 종합 점수가 74점에서 61점으로 하락했습니다.
- 특히 여드름, 붉은기, 건조 점수가 함께 떨어졌습니다.
- 최근 2주 로그를 보면 수면 부족, 스트레스 증가, 새 제품 사용의 영향이 크게 추정됩니다.
- 원인 기여도 예시: 수면 부족 0.33, 스트레스 0.27, 제품 변경 0.19

예시 2. 좋아진 경우

- 종합 점수가 65점에서 79점으로 상승했습니다.
- 특히 붉은기와 유분 균형 점수가 개선되었습니다.
- 최근 2주 로그를 보면 수면 안정, 자극성 제품 중단, 세안 루틴 유지의 영향이 크게 추정됩니다.
- 개선 기여도 예시: 규칙적 수면 0.29, 저자극 제품 사용 0.24, 수분 섭취 증가 0.18

핵심:

- `나빠졌다`면 악화 원인 설명
- `좋아졌다`면 개선 원인 설명
- 둘 다 `단일 원인 단정`보다 `기여도 순위`가 더 안전함

### 6-5. 실제 추론 파이프라인은 이렇게 동작하면 좋음

1. 오늘 얼굴 사진을 넣어 이미지 모델이 항목별 점수를 계산합니다.
2. 최근 14일 또는 30일의 과거 점수와 비교합니다.
3. 종합 점수 변화량이 임계값보다 작아지면 `악화`, 커지면 `개선`으로 판단합니다.
4. 어떤 세부 항목 점수가 가장 많이 변했는지 찾습니다.
5. 시계열 모델이 최근 생활습관/제품/환경 로그를 보고 원인 기여도를 계산합니다.
6. 멀티모달 모델이 `무엇이 나빠졌는지(또는 좋아졌는지)`와 `왜 그랬는지`를 하나의 설명으로 합칩니다.

짧게 말하면:

`사진 채점 -> 과거와 비교 -> 변화 감지 -> 원인 추정 -> 사람 말처럼 설명`

## 7. 이 프로젝트에서 평가해야 할 피부 요소

단순 여드름만 보면 프로젝트 목적과 맞지 않습니다.
최소한 아래 요소는 점수화 대상에 포함하는 것이 좋습니다.

### 필수 피부 요소

- 여드름
- 붉은기
- 건조
- 유분
- 색소침착
- 주름
- 모공

### 있으면 좋은 추가 요소

- 피부 결(texture)
- 민감도 관련 시각 징후
- 피부톤 균일도
- 광택/번들거림

### 추천 출력 형식

- 항목별 건강 점수: 0~100
- 종합 피부 점수: 0~100
- 최근 변화량: `+8`, `-12` 형태
- 위험 상태: 안정 / 주의 / 악화

## 8. A. 이미지 분석 멀티헤드 모델 설계

### 역할

얼굴 사진 1장을 보고 여러 피부 요소를 동시에 분석합니다.

### 비유

`한 명의 선생님이 한 장의 시험지를 보고 여러 과목을 한 번에 채점하는 것`과 같습니다.

### 입력

- 얼굴 사진 1장

### 전처리

- 얼굴 탐지
- 얼굴 정렬
- 피부 영역 분리
- 밝기/색상 보정
- 해상도 통일

### 출력 예시

- acne_score
- redness_score
- dryness_score
- wrinkle_score
- pigmentation_score
- oiliness_score
- pore_score
- overall_skin_score

### 추천 모델 구조

- Backbone: `EfficientNet-B3` 또는 `ConvNeXt-Tiny`
- Head: 각 피부 요소별 예측 head
- 방식: multi-task learning

### 왜 멀티헤드가 좋은가

피부 요소들은 서로 완전히 따로 노는 게 아니라 조금씩 연결되어 있습니다.

예를 들면

- 유분 변화는 여드름과 연결될 수 있고
- 자극은 붉은기와 건조에 같이 영향을 줄 수 있습니다.

그래서 모델이 한 가지 항목만 보게 하는 것보다,
여러 항목을 함께 배우게 하면 더 현실적인 표현을 잡을 가능성이 큽니다.

### Colab MVP 추천

- 이미지 크기: 320 이상
- 배치 크기: Colab 메모리에 맞게 8~32 사이 조정
- 손실함수: 항목별로 classification/regression 혼합 가능
- 우선순위: `여드름, 붉은기, 건조, 색소침착, 주름` 먼저 안정화

## 9. B. 시계열 원인 분석 모델 설계

### 역할

최근 며칠~몇 주의 로그를 보고
피부가 왜 나빠졌는지 또는 왜 좋아졌는지 추정합니다.

### 비유

`탐정이 CCTV를 시간 순서대로 돌려보며 사건 전후를 분석하는 것`과 같습니다.

### 입력 예시

- 날짜별 수면 시간
- 날짜별 물 섭취량
- 날짜별 당류 섭취 빈도
- 날짜별 유제품 섭취 빈도
- 날짜별 운동량
- 날짜별 스트레스 점수
- 날짜별 마스크 착용 시간
- 날짜별 햇빛 노출
- 날짜별 제품 사용 여부/변경 여부
- 날짜별 환경 정보(온도, 습도, 미세먼지)
- 날짜별 이미지 모델 점수

### 출력 예시

- 7일 후 악화 확률
- 14일 후 악화 확률
- 최근 악화 원인 top-k
- 최근 개선 원인 top-k
- 생활습관 개입 우선순위

### 추천 모델 구조

- MVP: `GRU + attention`
- 이후 확장: `LSTM`, `Transformer Encoder`, `TFT`

### 왜 처음엔 GRU가 좋은가

Colab에서는

- 구현이 쉽고
- 메모리 부담이 덜하고
- 시계열 길이가 아주 길지 않을 때 충분히 강력하기 때문입니다.

즉 처음에는 `작지만 믿을 만한 탐정`을 쓰는 것이 낫습니다.

## 10. C. 멀티모달 원인 추론 모델 설계

### 역할

이미지 모델과 시계열 모델의 결과를 합쳐
"왜 이런 변화가 생겼는지"를 최종 판단합니다.

### 비유

`사진 증거`와 `생활기록부`를 함께 보는 회의실이라고 생각하면 됩니다.

### 입력

- 이미지 임베딩
- 시계열 임베딩
- 정적 정보
  - 성별
  - 연령대
  - 기본 피부 타입
  - 민감 피부 여부

### 출력

- 종합 피부 상태
- 악화/개선 여부
- 원인 기여도 순위
- 설명 문장 생성용 구조화 결과

### 추천 구조

- image encoder output
- temporal encoder output
- static feature embedding
- fusion block
- multi-task heads

### 쉬운 해석

여기서 `fusion`은
`여러 선생님의 점수를 한 장의 종합 성적표로 합치는 것`입니다.

즉 사진에서 본 사실,
생활습관에서 본 사실,
기본 피부 특성을 합쳐서
최종 결론을 만드는 단계입니다.

## 11. "원인"은 단정이 아니라 "기여도"로 설명해야 함

이 프로젝트에서 매우 중요한 원칙입니다.

피부 악화는 대부분 원인 하나로 끝나지 않습니다.

예를 들면

- 잠을 못 잤고
- 스트레스를 많이 받았고
- 자극적인 제품을 새로 썼고
- 햇빛도 많이 쬈을 수 있습니다.

그래서 아래처럼 말하는 것이 더 안전합니다.

- 수면 부족 기여도: 0.31
- 스트레스 증가 기여도: 0.26
- 제품 변경 기여도: 0.21
- 자외선 노출 증가 기여도: 0.12

즉,
`범인을 딱 한 명 지목하는 수사`가 아니라
`누가 얼마나 관련 있는지 순위를 매기는 수사`가 더 맞습니다.

이 원칙은 개선 설명에도 똑같이 적용됩니다.

- 수면 안정 기여도: 0.28
- 당류 감소 기여도: 0.19
- 보습 루틴 유지 기여도: 0.24

## 12. 핵심 용어를 중3도 이해할 수 있게 설명

### 멀티모달(Multimodal)

여러 종류의 정보를 함께 쓰는 방식입니다.

비유:
`사진, 글, 표를 한꺼번에 읽는 것`

이 프로젝트에서는

- 얼굴 사진
- 생활습관 로그
- 제품 사용 기록
- 환경 데이터

를 함께 봅니다.

### 시계열(Time Series)

시간 순서대로 쌓인 데이터입니다.

비유:
`하루하루 쓴 일기장`

### 백본(Backbone)

이미지의 중요한 특징을 뽑아내는 모델의 중심 몸통입니다.

비유:
`선생님의 기본 실력`

### 인코더(Encoder)

입력을 이해하기 쉬운 숫자 표현으로 바꾸는 부분입니다.

비유:
`긴 글을 핵심 메모로 요약하는 사람`

### 임베딩(Embedding)

모델이 이해할 수 있도록 정보를 숫자 벡터로 바꾼 결과입니다.

비유:
`복잡한 사람 설명을 학생부 한 줄 요약으로 바꾼 것`

### 헤드(Head)

최종 예측을 내는 출력 부분입니다.

비유:
`채점 결과를 적는 채점창`

### 멀티헤드(Multi-head)

출력 창이 여러 개인 구조입니다.

비유:
`한 선생님이 여러 과목 점수를 동시에 적는 것`

### 멀티태스크 학습(Multi-task Learning)

한 모델이 여러 문제를 동시에 배우는 방식입니다.

비유:
`영어, 수학, 과학을 함께 공부하는 학생`

### 세그멘테이션(Segmentation)

이미지에서 특정 영역만 따로 나누는 작업입니다.

비유:
`색칠공부에서 얼굴 피부 부분만 테두리 따기`

### 병변(Lesion)

피부에 나타난 이상 부위입니다.

비유:
`지도에서 표시된 문제 지점`

### 심각도(Severity)

증상이 얼마나 심한지입니다.

비유:
`가벼운 감기인지, 심한 감기인지 따지는 것`

### 어텐션(Attention)

모델이 중요한 부분에 더 집중하는 장치입니다.

비유:
`시험지에서 틀리기 쉬운 문제를 먼저 보는 눈`

### 퓨전(Fusion)

여러 정보를 합치는 과정입니다.

비유:
`반 친구들의 의견을 모아 최종 결론 내기`

### 기여도(Attribution / Contribution)

각 원인이 결과에 얼마나 영향을 줬는지 수치로 나타낸 것입니다.

비유:
`축구 경기에서 각 선수가 승리에 얼마나 기여했는지 보는 것`

### 일반화(Generalization)

학습 때 본 사진 말고 처음 보는 사진에서도 잘 작동하는 능력입니다.

비유:
`학교 시험만 잘 보는 게 아니라 실전에서도 잘하는 실력`

### 데이터 편향(Bias)

특정 피부톤, 특정 환경, 특정 연령 사진만 많아서 모델이 한쪽으로 치우치는 문제입니다.

비유:
`한 반 학생만 보고 전교생을 판단하는 것`

## 13. 피부 점수 시스템 제안

사용자에게 보여줄 때는 아래처럼 정리하는 것이 좋습니다.

### 항목별 건강 점수

- 여드름 건강 점수
- 붉은기 건강 점수
- 건조 건강 점수
- 주름 건강 점수
- 색소침착 건강 점수
- 유분 균형 점수
- 모공 점수

점수 범위는 `0~100`, 높을수록 좋은 방향으로 맞춥니다.

### 종합 피부 점수 예시

초기 MVP에서는 단순 가중합으로 시작할 수 있습니다.

예시:

`종합 점수 = 0.25*여드름 + 0.15*붉은기 + 0.15*건조 + 0.10*주름 + 0.15*색소침착 + 0.10*유분균형 + 0.10*모공`

주의:

- 이 가중치는 정답이 아니라 초기값입니다.
- 사용자 목표에 따라 가중치를 바꿀 수 있습니다.
- 예를 들어 어떤 사용자는 여드름이 가장 중요하고, 어떤 사용자는 건조/붉은기가 더 중요할 수 있습니다.

## 14. 데이터 전략: 공개 데이터 + 자체 수집 데이터

이 프로젝트는 **공개 데이터만으로 완성되기 어렵습니다.**

왜냐하면 진짜 필요한 데이터는
`한 사람의 사진 + 생활습관 + 제품 사용 + 환경 변화 + 장기적인 피부 변화`
가 시간 순서대로 모두 연결된 형태이기 때문입니다.

그런데 이런 공개 표준 데이터는 매우 드뭅니다.

그래서 현실적인 전략은 아래와 같습니다.

### 1단계

공개 이미지 데이터로 이미지 모델 베이스라인 학습

### 2단계

앱/설문/로그 기반으로 자체 사용자 종단 데이터 수집

### 3단계

이미지 + 생활습관 + 제품 이력 기반 멀티모달 모델 고도화

쉽게 말해,
처음에는 `기출문제집`으로 공부하고,
나중에는 `우리 반 학생들의 실제 수행평가 기록`을 모아 더 똑똑해지는 구조입니다.

각 데이터셋이 어디에 도움이 되는지 역할을 나눠 쓰는 것이 좋습니다.

### 15-1. 핵심 여드름/피부 이미지 데이터

#### PAD-UFES-20

- 스마트폰 기반 임상 피부 이미지와 표형 변수 결합 사례

## 16. 추가로 추천하는 데이터셋

사용자 요청의 "더 좋은 데이터가 있으면 추천"에 대해,
현재 방향에서 특히 보강 가치가 있는 데이터는 아래와 같습니다.

### Fitzpatrick17k

- 다양한 피부 질환과 피부톤 정보가 있는 대형 피부 이미지 자원
- 모델이 특정 피부톤에만 치우치지 않도록 일반화와 공정성 측면에서 도움
- 여드름 전용은 아니지만, 피부톤 다양성 보강용으로 유용

### SKINCON

- 피부 질환 자체보다 `설명 가능한 개념(concept)`이 잘 정리된 데이터셋
- 예: scale, plaque, erosion 같은 사람 눈으로 이해 가능한 개념 라벨
- 이 프로젝트에서 "왜 그렇게 판단했는지"를 연구할 때 큰 도움
- 즉 `설명 가능한 AI` 방향 보강용 데이터

### FFHQ-Wrinkle

- 주름 segmentation/학습에 도움이 되는 공개 자원
- 여드름 외의 `주름` 요소를 다루려면 꽤 유용
- 다만 일반 얼굴 데이터 기반이라 피부 질환 데이터와는 목적이 조금 다름

## 17. 데이터셋 사용 전략을 현실적으로 정리하면

### 이미지 모델용

- ACNE04 / ACNE04-v2: 여드름 핵심 학습
- SCIN: 일반화 보강
- DDI / DDI-2: 피부톤 다양성 보강
- Fitzpatrick17k: 피부톤 및 외부 일반화 보강
- FFHQ-Wrinkle: 주름 보조 학습

### 설명가능성 보강용

- SKINCON: 사람이 이해할 수 있는 피부 개념 학습 참고

### 멀티모달 구조 참고용

- PAD-UFES-20: 이미지 + 표형 정보 결합 아이디어 참고

### 반드시 따로 구축해야 하는 것

- 날짜별 생활습관 로그
- 날짜별 제품 사용 로그
- 날짜별 환경 로그
- 사용자별 장기 추적 데이터

### 여러 데이터셋을 함께 쓸 때의 실전 규칙

여기서 매우 중요한 점은 `모든 데이터셋이 모든 라벨을 다 가지고 있지 않다`는 것입니다.

예를 들어,

- 어떤 데이터셋은 여드름 라벨만 강하고
- 어떤 데이터셋은 피부톤 다양성이 강하고
- 어떤 데이터셋은 주름에 강하고
- 어떤 데이터셋은 설명용 개념 라벨이 강합니다.

그래서 실전에서는 아래처럼 학습하는 것이 좋습니다.

1. 공통 라벨 사전을 먼저 만든다.
   - 예: acne, redness, dryness, wrinkle, pigmentation, oiliness, pore

2. 없는 라벨은 억지로 채우지 말고 `missing label`로 둔다.

3. 학습할 때는 `해당 라벨이 있는 샘플만 그 head의 loss를 계산`한다.

4. 데이터셋 이름 자체를 메타정보로 넣어 domain shift를 관리한다.

5. 마지막에는 자체 수집 데이터로 실제 서비스 환경에 맞게 다시 미세조정(fine-tuning)한다.

쉽게 말하면,
`수학 문제집, 영어 문제집, 과학 문제집을 한 번에 섞어 공부하되, 없는 문제까지 억지로 푸는 건 아니다`라는 뜻입니다.

즉,
공개 데이터는 `출발선`,
자체 수집 데이터는 `진짜 경쟁력`입니다.

## 18. 이 프로젝트에서 가장 현실적인 모델 조합

### A. 이미지 분석 멀티헤드 모델

- Backbone: `EfficientNet-B3` 또는 `ConvNeXt-Tiny`
- Output: 여드름, 붉은기, 건조, 주름, 색소침착, 유분, 모공, 종합 점수

### B. 시계열 원인 분석 모델

- 시작: `GRU + attention`
- 확장: `LSTM`, `Transformer Encoder`, `TFT`

### C. 멀티모달 원인 추론 모델

- image embedding + temporal embedding + static embedding
- fusion block
- multi-task outputs

이 조합이 좋은 이유는

- Colab에서 학습 가능하고
- 너무 무겁지 않으며
- 공모전/연구 MVP로 설득력이 충분하기 때문입니다.

## 19. 최종 출력 예시

사용자에게는 아래처럼 보여주는 것이 이해하기 쉽습니다.

### 예시 1. 악화된 경우

- 종합 피부 점수: 74 -> 61
- 상태: 악화
- 크게 나빠진 항목: 여드름, 붉은기, 건조
- 원인 후보 기여도:
  - 수면 부족 0.33
  - 스트레스 증가 0.27
  - 새 제품 사용 0.19
- 권장 행동:
  - 수면 시간 회복
  - 자극 제품 중단
  - 보습 루틴 강화

### 예시 2. 좋아진 경우

- 종합 피부 점수: 65 -> 79
- 상태: 개선
- 크게 좋아진 항목: 붉은기, 유분 균형
- 개선 기여도:
  - 규칙적 수면 0.29
  - 세안 루틴 안정화 0.21
  - 수분 섭취 증가 0.18
- 유지 권장 행동:
  - 현재 루틴 유지
  - 자극적인 제품 교체 피하기
  - 야외 활동 시 자외선 관리 지속

## 20. 지금 단계에서 꼭 기억해야 할 한계

### 1. 사진만으로 모든 원인을 100% 맞힐 수는 없음

사진은 `결과`를 잘 보여주지만,
원인은 생활습관과 시간 흐름 데이터가 있어야 더 잘 추정됩니다.

### 2. 공개 데이터만으로는 장기 원인 추론이 부족함

진짜 핵심 경쟁력은 결국 자체 종단 데이터입니다.

### 3. 여러 데이터셋을 합치면 라벨 기준이 다를 수 있음

예를 들면

- 어떤 데이터는 여드름 중심
- 어떤 데이터는 피부톤 중심
- 어떤 데이터는 개념 설명 중심
- 어떤 데이터는 스마트폰 임상 사진 중심

이라서, 하나의 공통 라벨 체계로 재정리해야 합니다.

### 4. 설명은 "의학적 확정 진단"이 아니라 "모델 기반 추정"이어야 함

즉 앱이 말해야 하는 것은
`확정 진단`이 아니라
`가능성이 높은 원인 후보와 기여도`입니다.

## 21. 최종 결론

이 프로젝트의 본질은
**"피부 사진 분류기"가 아니라**
**"피부 변화의 이유를 시간 흐름 속에서 설명하는 멀티모달 피부 코치"**입니다.

현재 개발 단계에서는 아래 3개가 핵심입니다.

1. 이미지 분석 멀티헤드 모델
2. 시계열 원인 분석 모델
3. 멀티모달 원인 추론 모델

그리고 사용자 요청사항을 반영하면,
시스템은 반드시 아래처럼 동작해야 합니다.

1. 여드름만이 아니라 주름, 건조, 붉은기, 색소침착 등 여러 요소를 함께 점수화한다.
2. 점수가 떨어졌다면 왜 나빠졌는지 설명한다.
3. 점수가 올랐다면 왜 좋아졌는지 설명한다.
4. 설명은 단정형보다 기여도 기반으로 제시한다.
5. 공개 데이터로 출발하되, 최종 경쟁력은 자체 장기 데이터에서 만든다.
